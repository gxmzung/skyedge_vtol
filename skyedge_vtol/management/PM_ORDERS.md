# PM Orders

## 공통 원칙
1. PX4는 바꾸지 않는다.
2. ROS2 상위 스택만 자체개발한다.
3. 시뮬에서 안 되는 기능은 실기체에 올리지 않는다.
4. 새 기능보다 재현성을 우선한다.
5. 모든 노드는 입력 / 출력 / 실패조건을 문서화한다.

---

## 팀 전체 오더
- 이번 시즌 목표는 **1등 가능한 자율임무 소프트웨어 완성**이다.
- 작년처럼 하부 플랫폼을 늦게 바꾸는 일은 없다.
- FC는 PX4로 고정한다.
- 각 파트는 자기 코드보다 **전체 임무 성공 흐름**에 맞춘다.
- 발표용 말보다 **로그와 재현성**을 먼저 만든다.

---

## 파트별 오더

### PX4 / Bridge 파트
- 이번 주 안에 SITL + QGC + ROS2 연결 성공
- arm / offboard / takeoff / land를 반복 가능하게 만들 것
- 토픽 목록과 command 흐름을 문서화할 것

### Vision 파트
- YOLO 금지, 라인과 ArUco 최소형부터
- 검출 결과는 숫자화해서 publish
- 카메라 입력이 없으면 fallback 상태를 명시할 것

### Guidance 파트
- hover, line-follow, marker-align, return-home 네 모드만 우선 구현
- 미션 상태와 분리해서 설계
- 하드코딩 줄이고 yaml 파라미터화

### Integration / Test 파트
- 테스트는 데모가 아니라 반복 성공률로 평가
- launch 실패, 토픽 누락, timeout을 먼저 잡을 것
- 실패 원인은 감으로 말하지 말고 로그로 적을 것

### Docs / QA 파트
- 설치 문서는 새 팀원이 30분 안에 따라갈 수 있어야 함
- 심사 대응 문서는 “우리가 직접 만든 부분” 중심으로 정리
- 질문 리스트와 방어 답변 초안 작성
